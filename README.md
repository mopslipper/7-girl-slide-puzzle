## 7 Girl Slide Puzzle

Astro 5 で作るスライドパズルです。ポートフォリオサイトと同じテーマ色を使い、作品画像は jsDelivr 参照で読み込みます。

## 開発

```bash
npm install
npm run dev
```

## テスト

```bash
npm test
```

## デプロイ

GitHub Pages 向けに `base='/7-girl-slide-puzzle'` でビルドします。
git commit -m "docs: add CLI and documentation"
```

## 🎓 TDDの原則（このテンプレート用）

### やること ✅

1. **仕様はテストで表現** - 曖昧な文章より実行可能なコード
2. **テストを先に固定** - 実装がテストに合わせる（逆ではない）
3. **小さく反復** - 1回で全部作らず、段階的に
4. **抜け道を検出** - テストだけ通す実装を防ぐ
5. **一般化を検証** - サンプル以外でも動くか確認
6. **スキル単位で責務分離** - 1スキル1責務で検証可能にする

### やらないこと ❌

1. **テストを実装に合わせない** - 仕様（テスト）は固定
2. **過剰なテストを書かない** - 実装詳細に踏み込まない
3. **一度に大きな変更** - 小さく反復で収束させる
4. **抜け道を放置しない** - ハードコードは必ず検出・修正

## 💡 Tips

### テストが収束しない時

1. **仕様が曖昧** → テストに仕様を追加
2. **入力領域が広すぎる** → 例を増やす
3. **設計上の決めごと不足** → エラー仕様、丸め、順序などを明確化

### 良いテストの書き方

```python
# ❌ 悪い例: 実装詳細に依存
def test_internal_function_called():
    assert module._internal_function.call_count == 3

# ✅ 良い例: 入出力に注目
def test_output_format():
    result = module.process("input")
    assert isinstance(result, str)
    assert len(result) > 0
```
